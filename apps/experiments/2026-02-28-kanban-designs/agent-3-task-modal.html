Description

The sandbox file system watcher needs to monitor all file changes made by coding agents in real-time and broadcast them to the UI via Convex subscriptions.

Requirements

  • Watch /workspace directory recursively for creates, updates, and deletes
  • Debounce rapid changes (50ms window) to prevent flooding
  • Emit structured events with path, action, and timestamp
  • Support .gitignore-style exclude patterns for node_modules

Implementation Notes

Use the Daytona SDK's fs.watch() API with the recursive flag. The watcher should be started when an agent begins execution and cleaned up on completion.

const watcher = daytona.fs.watch("/workspace", {
  recursive: true,
  filter: ["!node_modules", "!.git"],
});

watcher.on("change", (event) => {
  ctx.runMutation(api.files.recordChange, {
    agentId,
    path: event.path,
    action: event.type,
  });
});
Edge case: handle symlinks carefully -- the watcher should resolve them but not follow circular references.
